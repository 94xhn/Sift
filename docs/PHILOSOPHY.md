# Sift 的立场：诚实，而非操纵

> 这份文档说明 Sift 为什么这样设计。它是产品的"为什么"，也是我们拒绝某些做法的声明。

## 人在刷短视频时，到底在焦虑什么

焦虑有两层。多数产品只盯第一层，Sift 的战场在第二层。

### 第一层：刷的当下（被平台设计出来的焦虑）
- **FOMO / 社会比较**（Festinger）：你刷到的是全人类的高光集锦，比较对象被算法挑到最优，必然产生相对剥夺感。
- **间歇性强化**（Skinner 变率强化）：短视频是老虎机，"下一条可能更好"让人停不下来。
- **注意力被商品化**（注意力经济）：你不是用户，是被卖的货；界面在对抗你"想停下"的意志，制造失控感。

### 第二层：刷完之后（真正的痛 —— Sift 的战场）
- **时间负罪感 / 意志软弱**（akrasia）：明知不该却还是做了，刷完那股空虚自责。
- **自我差异**（Higgins）：现实自我与"本该在学习"的应然自我之间的落差。
- **社会加速与时间荒**（Hartmut Rosa）：一切都在加速，人永远觉得落后；解药是"共鸣"，而非不断滑过。
- **功绩社会的自我剥削**（韩炳哲《倦怠社会》）：当代人自己压迫自己，连休息都背上"必须有产出"的债。

### 哲学总根：刷视频是一场"逃避"
- **帕斯卡（divertissement）**：人类的不幸源于"不能独自安静地待在房间里"，用消遣逃避直面自身处境。短视频是最完美的消遣。
- **海德格尔（沉沦 / 闲言）**：人沉入"常人"的闲言与分心，躲开对有限性的直面。信息流是闲言的工业化。
- **克尔凯郭尔（焦虑是自由的眩晕）**：刷完的不安，是你的自由在提醒你——这段时间是你"选"的。

> 最深的焦虑不在内容里，而在划走之后那几秒：
> **"我的有限生命，又有一段被我自己换成了什么都没有。"**

## 我们的原则：接住焦虑，不放大焦虑

放大焦虑必然失败 —— 不是道德问题，是机制问题：用户缓解焦虑最快的方式，是**卸载那个让他内疚的 App**。一个开始"鞭策"用户的工具，会变成功绩社会的又一个监工。

所以 Sift 把每种焦虑**接成一个缓解它的功能**：

| 焦虑 | Sift 的接法 | 缓解机制 |
|---|---|---|
| 时间负罪感 | 诚实仪表盘：模糊的"浪费了"→具体数字 + "但你捞到了 N 条" | 把弥散焦虑对象化 → 可控感 |
| 自我差异 | 知识沉淀 / 周报：给应然自我一个证据 | 缩小落差 |
| 丧失能动性 | 悬浮球这个动作本身：是**你**决定这条值得留 | 夺回控制点 |
| FOMO | 存下即安心："不会丢了" | 允许自己划过去 |
| 社会比较 | 指标只跟自己的成长曲线比 | 切断横向比较 |

## 我们拒绝做的事

- ❌ 不制造内疚循环（"你又浪费了 X 小时！"）来提高黏性。
- ❌ 不美化刷视频、不给"刷得心安理得"提供借口。
- ❌ 不把用户数据上报到任何 Sift 自有服务器。

## 我们敢做的事：必要时劝你放下它

最难被模仿的设计，是**一个会在恰当时刻让你别用它的产品**。当你久刷而零沉淀，仪表盘不会催你，而会说：

> "今天刷了很久，沉淀了 0 条。或许今天不必沉淀什么——直接休息也很好。"

（实现见 `core/domain/.../dashboard/HonestNudge.kt`。）

一个敢说这句话的产品，赢得的是最深的信任。这就是"数据说话"的终极形态：**一面诚实的镜子，而不是又一个监工。**
